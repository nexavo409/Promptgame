# Prompt Architect — プロンプトTCG (MVP)

設計書「プロンプトトレーディングカードゲーム」に基づくMVP（最小プレイ可能プロトタイプ）です。
プレイヤーがカードでプロンプトを組み立て、AIが実行・採点して勝敗を決します。

## 動かし方

このプロジェクトは依存関係ゼロの静的Webアプリです。ES Modulesを使用しているため、
`file://` ではなく簡易HTTPサーバ経由で開いてください。

### Python があれば

```powershell
# プロジェクトディレクトリで
python -m http.server 5173
```

ブラウザで http://localhost:5173/ を開きます。

### Node.js があれば

```powershell
npx --yes http-server -p 5173 -c-1
```

### APIキー設定（任意）

ヘッダ右側に Anthropic APIキーを入力して「保存」すると、実際のClaudeでプロンプトの実行と
採点が行われます（`localStorage` に保存）。未設定の場合は決定論的なモック採点で遊べます。

> セキュリティ注意: ブラウザ直叩きのため `anthropic-dangerous-direct-browser-access` ヘッダを
> 付与しています。これはローカル単体プレイ目的での割り切りです。公開サイトには絶対に
> APIキーを含めないでください。

## 遊び方

1. ホーム画面でプリセットデッキ（4種）から選ぶか、デッキビルダーで自作する。
2. 「対戦」を押すと初期手札7枚が配られ、必要に応じてマリガン。
3. お題が出題されたら、手札からタスク1枚以上・構造1枚以上・コスト合計10以内でカードを選択。
4. 「構築を確定して実行」でAIが両者のプロンプトを実行・採点。
5. 合計点で勝った方がラウンド勝利。3ラウンド先取で勝利。

### 採点の仕組み

- AIが3軸（accuracy / utility / novelty）を各0-10点で採点。
- カードの**カテゴリ相性**: お題と一致するカードは効果1.5倍、不一致は0.7倍、汎用は1.0倍。
- **シナジー**: 特定の組み合わせで追加点（経営層ブリーフィング、データ可視化など8種）。
- **難易度補正**: 高難易度のお題は最終合計に1.3倍。
- 同点時は使用カード枚数が少ない方が勝利（効率性ルール）。

```

  data/
    cards.js            全カードプール（60+枚）

  game/
    rules.js            ゲーム定数・デッキ検証

  ai/
    client.js           Anthropic API呼び出し + モックフォールバック
  ui/
    app.js              UIコントローラ
```

## 既知の制限

- 実プレイヤー2人対戦（オンライン）は未実装。AI対戦のみ。
- アカウント・ランキング・経済システムは未実装。
- ユーティリティカードは **1ラウンドにつき1枚** 発動可能（コストは10枠から消費）。
  知識探索／完璧な準備＝山札上位を見て選ぶピックモーダル、思考整理＝手札を選んで捨て同数引く、
  エラー回避＝精度+1.0／優先度最高＝有用+0.6 独自+0.4 のスコアブースト＋プロンプト指示として反映。
- AI対戦相手も同様に1ラウンドで0〜1枚のユーティリティを発動します（エラー回避／優先度最高のみ自動選択）。
- AI判定はモデル応答のばらつきを含みます。重要な評価には複数回試行を推奨。
